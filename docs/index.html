🔍 Model Trust Scorecard

Transparent, reproducible verification of AI model benchmark claims

The dashboard is the result page.

When evaluations run, the published rankings appear on this site. Until then, this page explains where each output lands and how to request a review for a model that is not yet in the catalog.

Where the results go

Published dashboard

The current rankings are rendered into docs/index.html and served from GitHub Pages at this URL.

Machine-readable aggregate

The workflow writes the combined scores to trust_scores.json.

Markdown rankings

A repo-friendly table is also committed to trust_scores.md.

Per-run artifacts

Each workflow run uploads per-model JSON reports as GitHub Actions artifacts so the underlying verification output can be downloaded.

How to ask for a review of a model not shown here

1. Open a Model Submission issue

  • Use the Model Submission template.
  • Include the model name, vendor, benchmark claims, source URL, and license.
  • This is the clearest path if the model is not already in models/.

2. Submit a catalog entry by PR

If you already have the source details, add models/<model-id>.json in a pull request so the workflow can pick it up in normal runs.

3. Run a one-off review yourself

The CLI can score pasted claims without a catalog entry by using trust-scorecard score ... --text or --text-file.

Once the first evaluation is published, this page is replaced with the live rankings dashboard.
